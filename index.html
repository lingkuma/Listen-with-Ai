Listen-with-Ai

浏览器麦克风 / 系统音频 / 录屏测试

这是一个纯静态站点，可直接部署到 Vercel、Netlify 或任意静态托管。 下面 3 个页面分别针对 Windows、Android、Safari 做差异化测试说明。

Windows 测试页

重点验证麦克风、录屏、标签页/系统音频轨道是否能进入网页。

Android 测试页

重点验证移动端麦克风与 getDisplayMedia 限制，观察是否返回 audio track。

Safari / iPhone / iPad 测试页

重点验证 Safari 的麦克风、录屏与“其他 App 音频不可得”的现实限制。

测试原则

  • 麦克风：通过 getUserMedia({ audio: true }) 检测。
  • 其他 App / 系统音频：通过 getDisplayMedia({ video: true, audio: true }) 观察是否返回音频轨。
  • 录屏能力：通过屏幕共享 + MediaRecorder 录制结果验证。

注意：网页端无法直接越权抓取“其他 App 正在播放的声音”。本项目测试的是：浏览器在录屏授权场景下，是否愿意返回可用的音频轨。
